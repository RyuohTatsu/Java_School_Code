Secure Coding Overview

Secure coding is the principle of designing code that adheres to code security best practices (Michali, 2022). There are many tools available that help developers keep security in mind as they are writing the code. This is a big part of what the “Shift-left” security concept is about. These tools can be installed in your IDE and point out where a vulnerability might be while the developer is writing the code so you do not need to wait until the end to find the vulnerability.

This paper will look at three Security Checkers specifically for the Python language. (There are more security checkers that have multi-language support.) I will then choose one of the security checkers and analyze some Python code. I will discuss the results and the best practices to fix the vulnerabilities that were found.
