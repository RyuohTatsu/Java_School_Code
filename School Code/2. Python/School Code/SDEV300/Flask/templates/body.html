Python-Specific Security Checkers

Bandit

Bandit was developed by the Python Code Quality Authority (PyCQA) ( The Open-Source Code Security Tool RoundUp, 2023). Bandit was designed to find common security issues. It does this by processing your .py file and builds an AST. It then runs plugins against the AST nodes to find vulnerabilities. When it is finished it will generate a report to tell the developer the results ( Welcome to Bandit — Bandit Documentation, n.d.).

Pyre

Pyre was developed by Meta as open source ( The Open-Source Code Security Tool RoundUp, 2023). Pyre was designed to perform incremental checks which will let the developer know right away if there is a problem in the code (Pyre | Pyre, n.d.). It is easy to install into your IDE.

Safety CLI

Safety CLI was developed by Safety CLI Cybersecurity Inc. It uses the Python vulnerability database Safety DB ( The Open-Source Code Security Tool RoundUp, 2023). Safety CLI can be used on developer machines to scan individual requirements files or perform system-wide scans (Safety CLI 3 - Python Vulnerability Scanner, n.d.). Safety CLI will recommend best security practice fixes to most of the vulnerabilities it finds to save the developer’s time.

Although, using more than one security checker will help because any one checker is not 100% accurate.

The code also has a matrix game that uses numpy to calculate user input matrixes.

Explanation of the Process

I use VS code as my IDE. I installed Bandit and ran it on the .py file in my directory.

Results of Code Analysis

Summary of Results

The outcomes of the assessment deviated from my initial anticipations. While I foresaw security alerts related to outdated hashes, the evaluation unveiled a more concerning issue: the presence of hardcoded passwords across multiple instances. Although these hardcoded passwords may not pose an immediate high-severity threat, their existence introduces a vulnerability that could potentially lead to security breaches if exploited.

Upon closer examination, I discerned that the root of this vulnerability lay in the manner in which I assigned passwords. I inadvertently stored passwords as plain text within open strings. This oversight not only violates fundamental security principles but also increases the likelihood of unauthorized access should these passwords be exposed.

Discussion on Vulnerability Fixes

To address the vulnerabilities identified within the codebase, specific fixes are imperative. Firstly, tackling the issue of outdated hashes involves a meticulous review of the hashing algorithms employed. Outdated hashes, susceptible to various forms of attacks, necessitate an immediate upgrade to more robust and contemporary hashing techniques. This entails replacing deprecated algorithms like MD5 or SHA-1 with modern counterparts such as SHA-256 or bcrypt, ensuring a higher level of cryptographic security. On the other hand, mitigating the risk posed by hardcoded passwords demands a fundamental shift in approach. Rather than storing passwords directly within the source code, a more secure strategy involves dynamically generating passwords at runtime using a cryptographically secure random password generator. These dynamically generated passwords can then be securely managed and stored using encryption or other secure storage mechanisms, safeguarding sensitive credentials from unauthorized access or compromise. By implementing these fixes, the application fortifies its defenses against potential security threats, fostering a more robust and resilient security posture.

Conclusion

In conclusion, the evaluation of Python-specific security checkers, particularly Bandit, Pyre, and Safety CLI, has shed light on the critical role of secure coding practices in safeguarding software applications against potential security vulnerabilities. These tools serve as invaluable assets in the development process, enabling developers to identify and address security issues proactively. By selecting Bandit for code analysis, I was able to obtain insightful reports on security findings, facilitating timely remediation efforts. However, it is essential to acknowledge that no single security checker is infallible, emphasizing the importance of employing multiple tools and adopting a "shift-left" security approach to detect vulnerabilities early in the development lifecycle.

Furthermore, the analysis of the code revealed unexpected vulnerabilities, including the presence of hardcoded passwords, highlighting the significance of thorough code review and adherence to best security practices. The discussion on vulnerability fixes underscored the need for robust measures such as upgrading outdated hashes and implementing secure password management techniques. These measures not only enhance the resilience of the application but also mitigate the risk of unauthorized access and data compromise.

In essence, by integrating security checks into the development workflow and prioritizing the adoption of secure coding principles, developers can bolster the security posture of their applications and safeguard against potential threats. Moving forward, continued vigilance, regular updates, and ongoing education on emerging security trends will be crucial in maintaining a strong defense against evolving cyber threats.
